
CONTRIBUTING

Thank you for your interest in contributing to this project. 
GYMPLY is built on simplicity, privacy, and a local‑first philosophy. 
Contributions that respect these principles are highly appreciated.

🤝 What We Welcome
- Enhancements that extend functionality without adding unnecessary complexity
- Improvements to performance, stability, or code quality
- UI/UX refinements that strengthen the minimalistic design
- Bug fixes, documentation updates, and small improvements
- Contributions that avoid external trackers, analytics, or cloud dependencies

🚫 What We Avoid
- Features that violate the local‑only philosophy
- Heavy frameworks, unnecessary libraries, or vendor lock‑in
- Functionality that makes the app slower, heavier, or less minimalistic

🛠️ How to Contribute

1. Open an Issue
Use issues for:
- Bug reports
- Feature requests
- Design discussions
- Implementation questions
Please include:
- Steps to reproduce
- Screenshots or logs when relevant
- Platform and version details

2. Fork and Branch
- Fork the repository
- Create a new branch with a clear name, for example:
feature/add-1rm-calculator
fix/pwa-update-flow
docs/improve-readme
- Keep commits small and focused, with clear commit messages.

3. Submit a Pull Request
A good pull request includes:
- A short description of what and why
- Screenshots or GIFs for UI changes
- No breaking changes without prior discussion
- No new dependencies unless strongly justified
Pull requests are evaluated on:
- Simplicity
- Code quality
- Consistency with the existing style
- Respect for privacy and local‑only data handling

🧪 Testing
Ensure all existing functionality continues to work.
Add tests where appropriate (if a test framework is present).
Manual testing is acceptable as long as it is clearly described.

🧭 Code Style Guidelines
- Follow the existing structure and conventions
- Keep functions small and focused
- Avoid unnecessary abstraction
- Document only where it adds real value

🌱 Code of Conduct
We expect respectful communication and collaboration.
Discussions are welcome, but always constructive and focused on improving the project.

📄 License
By contributing, you agree that your contributions fall under the same license as the project.
